Distance matrix calculations can quickly become a bottleneck for a workflow because of their O(n^2) runtime complexity. This is pretty annoying, so I want to create a drop-in replacement for the scipy.spatial.distance.pdist function, which runs much faster. In this post, I am going to propose different impementations for calculating the distance matrix, predict the relative performance of each implementation, check if my predictions are correct and try to understand why they were wrong, as I am sure they will be.

What is a distance matrix?

A distance matrix is a matrix, where each element represents the distance between two points. A picture says a thousand words, so hopefully the interactive graphic will explain just how a distance matrix comes together.

As you can see, the matrix is symmteric, and each distance appears twice. This should come as no surprise, the distance from point a to point b is the same as the distance from point b to point a!
